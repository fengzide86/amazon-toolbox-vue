# 全量检查与部署 Skill

## 触发场景
- 修改代码前
- 修改代码后
- 部署到服务器前

## 核心原则
**改代码要有整体性思维，不要只改单一模块而忽略关联**

---

## 流程

### 第一阶段：改代码前 - 全面检查

1. **检查项目结构**
   - 列出所有目录和文件
   - 理解项目架构

2. **找出关联模块**
   - 谁调用了这个模块？
   - 这个模块依赖了谁？
   - 修改后是否影响上下游？

3. **列出修改计划**
   - 需要修改哪些文件？
   - 每个文件的修改内容是什么？

### 第二阶段：改代码时 - 同步修改

1. **同时修改所有关联文件**
   - 不要只改一个文件
   - 确保所有依赖都更新

2. **检查跨模块一致性**
   - 前端 API 路径与后端路由是否匹配？
   - 数据库模型与 schemas 是否匹配？
   - 导入路径是否正确？

### 第三阶段：改代码后 - 验证一键操作

1. **检查一键启动.bat**
   - 是否能正常启动？
   - 依赖是否完整？

2. **检查一键打包.bat / build.bat**
   - PyInstaller 的 hidden-import 是否包含所有新模块？
   - 打包后是否能正常运行？

3. **检查一键发布.bat / publish.bat**
   - 上传逻辑是否正常？
   - 服务器地址是否正确？

4. **检查 deploy.sh**
   - 是否需要同步更新？
   - 内嵌代码是否是最新版？

### 第四阶段：部署到服务器

1. **运行部署脚本**
   ```bash
   python deploy_to_server.py
   ```

2. **验证部署**
   - 健康检查：`curl http://8.130.113.104:8000/api/health`
   - 服务状态：`systemctl status toolbox-backend`

## 常见问题

### Q: 打包后运行报错找不到模块？
A: 检查 build.bat 和一键打包.bat 的 hidden-import 是否包含所有新模块。

### Q: 部署后服务启动失败？
A: 检查端口是否被占用，先杀掉旧进程再重启。

### Q: 前端调用 API 报错？
A: 检查前端 api.js 的路径是否与后端路由匹配。
